# ADR-0014: 予算・持ち越しモデル設計方針

- 日付: 2026-04-07
- ステータス: 決定済み

---

## コンテキスト

フェーズ2の予算管理機能に向けて、以下の設計方針を決定する必要があった。

- カテゴリ別の月次予算をどう管理するか
- 年単位で予算設定が変わる場合のバージョン管理
- 月末の差分（余剰/超過）を翌月へ持ち越す仕組み
- 前月が未確定の状態で今月を開いたときの挙動

---

## 検討した選択肢

### 予算のバージョン管理

**A. 月ごとに予算額を直接保存**
- シンプルだが、全カテゴリの金額を毎月コピーする必要がある
- 「今月から予算変更」の操作が煩雑

**B. テンプレート方式（採用）**
- 「2026年版」「転職後版」のように名前付きで保存
- 適用期間（valid_from / valid_to）で有効なテンプレートを自動選択
- 変更時は新テンプレートを作成して切り替える

### 持ち越しの確定方式

**A. 自動確定（月が変わったら自動計算）**
- 操作不要だが、「締め」の意識が生まれない
- 前月のデータを修正したときに遡及して変わってしまう

**B. 確定ボタン方式（採用）**
- 月末に明示的に確定することで数値が固定される
- 確定後は変更不可にすることでズレを防止
- 「今月はこれで締め」という家計管理の習慣に合う

### 未確定月の表示

**A. 持ち越し 0 円として表示**
- シンプルだが、前月を確定しないと今月の予算が正しく見えない

**B. 暫定計算値を表示（採用）**
- 前月未確定でも今月の画面をすぐ使える
- 「※暫定」バッジで確定値と区別
- 前月確定時に自動的に固定値に切り替わる

---

## 決定事項

### 1. テンプレート方式で予算をバージョン管理

**理由**
- 年単位・ライフイベント単位で予算を切り替えるユースケースに対応
- 毎月の予算コピー作業が不要
- どの月にどのテンプレートを使ったか履歴が残る

### 2. 月末確定ボタンで持ち越しを固定

**理由**
- 数値の遡及変更を防止
- 家計簿の「月締め」という自然な操作フローに合う

### 3. 未確定月は暫定計算値を表示

**理由**
- 確定を忘れていても今月の予算画面が使える
- 暫定バッジで状態が明確に分かる

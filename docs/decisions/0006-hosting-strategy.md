# ADR-0006: ホスティング・デプロイ戦略

- 日付: 2026-03-22
- ステータス: 決定済み

---

## コンテキスト

デプロイ先を決定する必要があった。
アプリは毎日使うものではなく、月数回CSVをアップして確認する程度の利用頻度。
ポートフォリオ用途も兼ねるため、採用活動時の見せ方も考慮が必要。

---

## 検討した選択肢

### コスト・cold start 比較（東京リージョン）

| | Fly.io | GCP | AWS |
|---|---|---|---|
| **月額（通常時）** | ~$0〜2 | ~$10 | ~$26 |
| **cold start** | 2〜5秒 | 1〜3秒（Cloud Run） | なし（常時起動） |
| **スケールゼロ** | ◎ | ◎（Cloud Run） | △（設定複雑） |
| **学習価値** | △ | ◎ | ◎ |
| **設定難易度** | 低 | 中 | 高 |

### Renderを選ばなかった理由

- cold startが30〜60秒と体感でわかるレベルで遅い
- ポートフォリオとして採用担当者に見せる際に「壊れてる？」と思わせるリスクがある

---

## 決定事項

### v1: Fly.io を採用する

**理由**:
- cold start 2〜5秒で許容範囲
- スケールゼロ対応でコストほぼ0（個人利用時）
- Dockerネイティブでローカルと本番環境が一致
- GitHub ActionsとのCI/CD連携がシンプル（`FLY_API_TOKEN`のみ）
- 採用活動時はアップグレードで常時起動に切り替え可能

**フェーズ別の運用**:

| フェーズ | 構成 | コスト |
|---|---|---|
| 開発中・個人利用 | Fly.io スケールゼロ + Fly PostgreSQL | ほぼ無料 |
| 採用活動時 | Fly.io アップグレード（常時起動） | 数百円〜/月 |

### v2以降: GCPを1ヶ月練習する（任意）

v1完成後に以下を候補として検討する：

- **GCP Cloud Run + Cloud SQL**（~$10/月）
  - Cloud Runはスケールゼロ対応、cold start 1〜3秒
  - `gcloud` CLI・IAM・サービスアカウントの学習が目的
  - 採用市場での「GCP経験あり」の価値

- **Next.js + Vercel 構成**
  - cold startほぼゼロ
  - v2（React + Go）の方向性に沿った技術チャレンジ

**AWSは現時点で対象外**: 最小構成で~$26/月と高コストのため費用対効果が低い。

# 明細カテゴリ分類設計

## 概要

CSVから取り込んだ明細を「食費」「交通費」などのカテゴリに分類する仕組み。
初回は未分類が多いため、手動補完 → AI 自動化の段階的な設計を採用。

---

## カテゴリの持ち方

カテゴリはユーザーごとに独立して保持する（テンプレートコピー方式）。

```
categories
  id
  user_id      NULL = システムテンプレート / NOT NULL = ユーザー独自
  name         食費・飲食費など（ユーザーが自由に命名）
  color        表示色
```

- 新規ユーザー登録時: システムテンプレートをコピーして `user_id` 付きで生成
- 以降はユーザーが自由に追加・編集・削除
- 他ユーザーのカテゴリには影響しない

---

## 店舗マッピングキャッシュ

店舗名単位でカテゴリをキャッシュし、AI コストを最小化する。

```
merchant_classifications
  id
  merchant_name      CSVに出てくる店舗名（例: "マクドナルド"）
  category_id        紐付けたカテゴリ
  user_id            NULL = 共有マッピング / NOT NULL = ユーザー個別
  source             ai / user_manual
  confidence         0.0〜1.0（AI の確信度）
  classified_at
```

### 分類の優先順位

1. `source = user_manual` のユーザー個別マッピング（最優先・上書き不可）
2. ユーザー個別の AI マッピング
3. 共有マッピング（他ユーザーが分類済みの店舗）
4. 未分類（手動割り当て待ち）

---

## フェーズ別実装計画

### フェーズ1: 手動分類

- 取り込んだ明細のうち未分類のものに「未分類」バッジを表示
- `merchant_name` が不明な場合（"PayPay" など）はユーザーが直接編集できる
- `merchant_name` を編集すると `merchant_classifications` を再検索して自動分類
- それでも未分類の場合はカテゴリを手動選択
- 割り当てると `merchant_classifications` に `source = user_manual` で保存
- 次回同じ店舗の明細は自動分類される

```
未分類明細
  → merchant_name 編集（"PayPay" → "ローソン"）
    → 分類済みならそのままカテゴリ確定
    → 未登録ならカテゴリ選択ドロップダウン → 保存
```

### フェーズ2: AI 深夜バッチ分類

- Fly.io cron で毎日深夜に実行
- `merchant_classifications` に登録のない店舗名を抽出
- Claude API に店舗名を渡してカテゴリを推定
- `source = ai`、`confidence` スコアとともに保存
- confidence が低い（例: 0.6 未満）場合は未分類のまま

### フェーズ3: 共有マッピング

- `user_id = NULL` の共有マッピングテーブルを整備
- 複数ユーザーが同じ店舗を同じカテゴリに分類 → 共有マッピングに昇格
- 新規ユーザーの初回取り込みが自動分類済みの状態になる

---

## UX フロー（フェーズ1）

```
CSV取り込み
  ↓
明細一覧表示
  ├─ 分類済み: カテゴリ名表示
  └─ 未分類: 「未分類」バッジ + カテゴリ選択ドロップダウン

ヘッダーに未分類件数バッジ（例: 「未分類 12件」）
→ 未分類一覧フィルタへのショートカット
```

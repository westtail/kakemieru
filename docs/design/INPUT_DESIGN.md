# インプット設計

最終更新: 2026-04-05

---

## 概要

KakeMieru へのデータ入力に関する設計・検討事項をまとめる。

---

## データの入力経路

| 経路 | フェーズ | ステータス |
|---|---|---|
| Web アップロード | フェーズ1 | 設計済み |
| CLI コマンド（`rails import:run`） | フェーズ4 | 設計済み |
| フォルダウォッチャー（常駐プロセス） | フェーズ4 | 設計済み |
| 手動入力（現金払い・CSVに載らない支出） | 未定 | **未検討** |

### 手動入力について
CSVに載らない支出（現金払い・交通系ICカードなど）をどう扱うか未決定。
対応するかどうかも含めて要検討。

---

## CSVフォーマット対応戦略

詳細は [decisions/0012-csv-import-strategy.md](decisions/0012-csv-import-strategy.md) を参照。

| フェーズ | 内容 |
|---|---|
| フェーズ1 | 特定1社を決め打ちで実装（対応カード会社は後日決定） |
| フェーズ2 | ヘッダー自動検出（列名キーワードマッチ） |
| 将来 | ヘッダーなしCSVのデータ形式から自動推定 |

---

## CSVの前処理

取り込み時に対応が必要な前処理。

| 項目 | 内容 | ステータス |
|---|---|---|
| 文字コード変換 | Shift-JIS → UTF-8 | 要実装 |
| BOM付きUTF-8 | Excelで保存したCSVに多い | **未検討** |
| 空行・コメント行のスキップ | 楽天CSVなど先頭に説明行がある場合あり | **未検討** |
| 金額のカンマ区切り | `1,000` を数値に変換 | **未検討** |
| 日付フォーマットの正規化 | `2026/01/01`・`2026-01-01`・`26/01/01` など | **未検討** |

---

## 重複取り込み防止

詳細は [decisions/0012-csv-import-strategy.md](decisions/0012-csv-import-strategy.md) を参照。

| ケース | 対応 |
|---|---|
| 同一ファイル（ハッシュ値一致）の2回目アップロード | エラーでブロック |
| 同日 + 同金額 + 同店舗の明細 | 警告を表示してユーザーに確認 |

---

## 取り込み後のデータ品質

| 項目 | ステータス |
|---|---|
| カテゴリ「未分類」になった明細を手動で修正 | **未検討** |
| 取り込んだ明細を削除・編集 | **未検討** |
| 取り込み履歴ログ（いつ何件取り込んだか） | **未検討** |

---

## カード・口座の管理

| 項目 | ステータス |
|---|---|
| 複数カードの管理（カードAとBを別々に管理） | 設計済み（PROJECT_OVERVIEW 参照） |
| デビットカード・銀行口座を対象にするか | **未検討** |

---

## 未検討事項まとめ

設計フェーズで決定が必要な項目：

- [ ] 手動入力（現金払い）に対応するか
- [ ] BOM付きUTF-8 への対応
- [ ] 空行・コメント行のスキップ処理
- [ ] 金額のカンマ区切り変換
- [ ] 日付フォーマットの正規化
- [ ] カテゴリ「未分類」明細の手動修正 UI
- [ ] 取り込み明細の削除・編集
- [ ] 取り込み履歴ログ
- [ ] デビットカード・銀行口座への対応
